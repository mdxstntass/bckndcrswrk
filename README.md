# CST3144 Backend

Simple Express + MongoDB API for lessons and orders.

## Files
- `index.js`: Express server with CORS, logging, static images, MongoDB connection helpers, routes for lessons search/list/update and order creation, and error handling.
- `.gitignore`: Ignores node modules, logs, env files, build/coverage output, and editor/OS cruft.
- `package.json`: Project metadata and scripts (`start`, `dev`) plus runtime deps (`express`, `mongodb`, `morgan`, `dotenv`) and dev dep `nodemon`.
- `package-lock.json`: Dependency lockfile generated by npm to pin exact versions.

## Getting Started
1) Copy `.env.example` to `.env` (or create `.env`) and set `MONGO_URL` and optional `DB_NAME`, `PORT`.
2) Install: `npm install`
3) Run dev (with reload): `npm run dev` — or prod: `npm start`
4) API base: `http://localhost:3000` (unless `PORT` overridden)

## Key Endpoints
- `GET /lessons` — list all lessons.
- `GET /search?q=...` — search by subject/location/description (partial, case-insensitive) or exact numeric `price`/`spaces`.
- `PUT /lessons/:id` — body `{ spacesDelta: number }` to increment/decrement available spaces.
- `POST /orders` — body requires `items`, `name`, `phone`; stores with `createdAt`.

## Notes
- Static images are served from `public/images` at `/images/...`.
- If MongoDB is not connected yet, requests return `503` until the connection succeeds.
